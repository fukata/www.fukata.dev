## コンテキスト

### レビュー対象のファイルパス
!`if [ -n "$ARGUMENTS" ] && [ -f "$ARGUMENTS" ]; then echo "$ARGUMENTS"; else ls -t _posts/*/*.md | head -1; fi`

### レビュー対象の記事
!`if [ -n "$ARGUMENTS" ] && [ -f "$ARGUMENTS" ]; then cat "$ARGUMENTS"; else echo "=== 最新の記事 ==="; LATEST=$(ls -t _posts/*/*.md | head -1); echo "ファイル: $LATEST"; cat "$LATEST"; fi`

### 参考用: 最近の別の記事（スタイル比較用）
!`for f in $(ls -t _posts/*/*.md | head -3 | tail -2); do echo "=== $f ==="; cat "$f"; echo; done`

### 過去に使用されたタグ一覧
!`grep -rh "^tags:" _posts/ | sed 's/tags: \[//;s/\]//' | tr ',' '\n' | sed 's/^ //' | sort -u`

### 全記事のタイトル一覧（相互リンク確認用）
!`grep -rh "^title:" _posts/ | sort`

## タスク

ブログ記事をレビューし、改善点をフィードバックしてください。

レビュー対象: `$ARGUMENTS`（未指定の場合は最新の記事）

### レビュー観点

以下の観点でレビューを行い、それぞれについて結果を報告してください。

#### 1. 誤字・文法チェック
- 誤字脱字の有無
- 助詞の誤用
- 不自然な表現
- マークダウン記法の誤り（閉じ忘れなど）

#### 2. 読みやすさ
- 段落の長さは適切か
- 文が長すぎないか
- 説明は分かりやすいか
- 専門用語に適切な説明があるか

#### 3. SEO観点
- タイトルは検索されやすいか（具体的な技術名・製品名が含まれているか）
- タグは適切か（過去のタグ一覧を参考に）
- タグが空の場合は適切なタグを提案する
- メタ情報（フロントマター）に不備がないか

#### 4. 構成・構造
- `##` 見出しが適切に使われているか
- `## まとめ` セクションがあるか
- 導入文が記事の内容を適切に要約しているか
- セクションの順序は論理的か

#### 5. スタイル一貫性
- このブログの文体と一致しているか（個人的・カジュアルな日本語）
- 過去の類似記事と比較してトーンが合っているか
- リンク形式が正しいか（過去記事への相対リンク、画像パスなど）

#### 6. フロントマター確認
- `layout: post` が設定されているか
- `date` のフォーマットが `YYYY-MM-DD HH:MM:00 +0900` か
- `tags` が配列形式 `[tag1, tag2]` か
- タイトルにコロンが含まれる場合、ダブルクォートで囲まれているか

### 出力形式

以下の形式でレビュー結果を報告してください:

```
## レビュー結果: [ファイル名]

### 総合評価
[A/B/C/Dの評価と一言コメント]

### 指摘事項

#### 誤字・文法
- [ ] 指摘1

#### 読みやすさ
- [ ] 指摘1

#### SEO
- [ ] 指摘1

#### 構成・構造
- [ ] 指摘1

#### スタイル一貫性
- [ ] 指摘1

#### フロントマター
- [ ] 指摘1

### 改善提案
[具体的な改善案]
```

指摘がない観点は「問題なし」と記載してください。

### 修正の適用

レビュー結果を報告した後、ユーザーに修正を適用するかどうか確認してください。ユーザーが同意した場合のみ、Edit ツールを使用してファイルを修正してください。
